# 铁路铁矿石运输预测：时间推演模块与业务语义解释（创新点说明）

## 一、创新点定位

在原有**直接预测输出数值向量**的时序模型（如 Time-LLM）基础上，增加一个**时间推演模块**：

- **输入**：历史真实数据 + 模型预测得到的未来数值序列  
- **过程**：将二者**融合**，在时间维度上做推演与关联分析  
- **输出**：面向铁路业务的**语义解释**（自然语言或结构化语义），而非仅给出预测数字  

即：**“数值预测 → 时间推演与融合 → 铁路业务逻辑的语义解释”**，使调度人员既能看到预测曲线，也能得到“为什么这样、业务上意味着什么、该如何应对”的说明。

---

## 二、整体架构与模块角色

```
历史多通道序列 ──┐
                 ├──→ [时序预测模型] ──→ 预测数值向量 (如 48 步 × 4 品种)
                 │
                 └──→ [时间推演模块] ←── 预测数值向量
                           │
                           ├ 融合：历史 + 预测（对齐时间、多通道）
                           ├ 推演：趋势、拐点、品种间关系、异常/风险时段
                           └ 输出：铁路业务逻辑的语义解释
```

- **时序预测模型**：负责从历史序列直接输出未来多步、多品种的数值（现有工作）。
- **时间推演模块**（创新）：不改变预测数值本身，而是**结合历史与预测**做融合与推演，并生成**可读、可操作的业务语义**，例如：
  - “未来两周铁矿石与铁矿砂呈交替发运，建议按日预留对应车皮”；
  - “铁精矿粉在预测中段明显回落，与常见检修窗口一致，可提前协调到端库存”；
  - “铁矿粉与铁精矿粉在预测期内走势相近，存在联合排产空间”。

这样，**预测结果**与**业务语义解释**一一对应，且解释建立在“历史 + 预测”的融合之上，而不是事后对数字的单独解读。

---

## 三、铁路业务背景与数据含义（支撑语义解释的领域）

### 3.1 四类货品与业务含义

| 指标       | 业务含义           | 与推演/解释的关系 |
|------------|--------------------|--------------------|
| **铁矿石** | 原矿/块矿粗料运输   | 运量大、波动高，语义中常涉及“主品类”“专列”“高峰/低谷” |
| **铁矿砂** | 砂状半成品         | 与铁矿石常交替发运，语义中可解释“品种间替代/竞争” |
| **铁矿粉** | 粉矿细料           | 量级与波动相对稳定，语义中可解释“平稳期”“排产窗口” |
| **铁精矿粉** | 精矿粉             | 与检修、固定客户相关，语义中可解释“计划性”“低谷原因” |

数据特性：日度、多通道、大量为 0（批次/专列）、多品种共享运力。时间推演模块可据此生成**与调度场景一致**的语义（如“某日某品种为 0 与计划排班一致”等）。

### 3.2 语义解释可覆盖的维度（示例）

- **趋势与拐点**：未来 48 天内各品种何时升、何时降，与历史近期是否一致。
- **品种间关系**：哪些品种“此起彼伏”、哪些“同升同降”，对应运力竞争或联合排产。
- **异常与风险**：预测中某段骤升/骤降或长期为 0，结合历史给出“可能对应检修/计划调整”等语义。
- **调度建议**：在语义中显式给出“建议预留车皮”“建议关注某品种某时段”等可操作表述。

上述维度均由**历史 + 预测融合后的时间推演**得到，而不是对单点数字的事后注释。

---

## 四、时间推演模块的输入输出与实现思路

### 4.1 输入

- **历史序列**：当前用于预测的输入窗口（如过去 96 天 × 4 品种），可选包含时间戳、节假日等。
- **预测序列**：模型输出的未来多步向量（如 48 天 × 4 品种），与历史在时间上首尾相接。

融合时可按“同一时间轴”对齐：`[历史 96 步; 预测 48 步]` 形成一段**连续时间线**，供推演模块分析趋势、拐点与品种关系。

### 4.2 输出（铁路业务逻辑的语义解释）

- **形式**：自然语言段落或结构化条目（如“趋势”“品种关系”“风险点”“建议”）。
- **要求**：用铁路调度可理解的术语（车皮、专列、检修、品种替代、到端库存等），且**每条结论都能对应到“历史+预测”中的具体时段或品种**，避免空泛描述。

### 4.3 实现思路（可选方向）

1. **基于规则 + 统计**：对融合序列做趋势、差分、品种相关性等统计，再按预设模板生成语句（如“铁矿石在 t1–t2 日上升，与历史 30 日趋势一致”）。
2. **小模型/轻量头**：用融合后的序列（或压缩表示）作为输入，训练一个“序列 → 文本”的生成模型，并在训练时引入铁路业务短语与例句，使生成内容符合领域用语。
3. **大模型 + 结构化输入**：将历史与预测整理成表格或短摘要（如“过去 96 天四品种统计 + 未来 48 天预测值”），作为 prompt 输入大模型，要求其严格基于这些数字输出铁路业务语义解释，并限制幻觉（只允许基于给定数字推理）。

模块可单独评估：如对“趋势方向”“品种关系”等做人工或规则标注，与模块输出对比，衡量语义一致性与可用性。

## 五、与测试集表现的关系

- **数值预测**（如 Time-LLM 在测试集上的 MAE/RMSE）仍然按现有方式评估；分品种误差差异（如块矿/砂类误差大、粉类误差小）可用于**验证预测质量**，并可作为时间推演模块的**输入质量说明**。
- **时间推演模块**的评估侧重：在相同历史与预测下，生成的语义是否与真实业务逻辑一致、是否对调度有可操作建议；测试集上的预测数值可为该模块提供**稳定的输入**，用于离线评估与案例展示。

因此：**创新点 = 在原有“只输出数字”的基础上，增加“历史+预测融合 → 时间推演 → 铁路业务语义解释”的模块**；测试集表现用于说明预测阶段可靠，从而支撑“解释是基于可靠预测的推演”这一叙事。

---

## 六、论文/报告中的创新点表述建议

- **问题**：既有方法多仅输出未来运量的数值向量，缺乏与历史衔接、与业务逻辑对应的语义解释，调度人员难以直接理解“预测在业务上意味着什么”。
- **做法**：在时序预测模型之上，增加**时间推演模块**，将**历史数据与预测结果融合**，在时间维度上对趋势、拐点、品种关系与异常进行推演，并**生成铁路业务逻辑的语义解释**（如品种替代、检修窗口、车皮预留建议等），实现从“数值预测”到“可解释的业务语义”的衔接。
- **价值**：预测结果与业务解释一一对应，解释建立在“历史+预测”的融合推演之上，便于调度决策与系统可解释性；可进一步在铁路铁矿石多品种场景下，用测试集预测结果驱动时间推演模块，展示端到端的数值+语义输出。

如需，可在文档中补充“时间推演模块”的接口定义（输入输出格式）或与现有 `run_eval` 的衔接方式，便于实现时对照。
