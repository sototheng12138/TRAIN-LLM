# 消融实验：重编程模块（Reprogramming Module）

## 目的

验证「时间序列 Patch → 文本原型交叉注意力 → LLM」中，**重编程层**（利用文本原型做跨模态语义对齐）是否必要。去掉该模块后，时间序列仅通过线性映射进入 LLM，不再与语言模型的文本原型做交叉注意力。

## 完整 Time-LLM 流程（保留重编程）

1. Patch Embedding：时间序列切 patch 并嵌入为 `(B, L, d_model)`。
2. **重编程层**：以 patch 嵌入为 Query，以 1000 个文本原型（由 `mapping_layer(word_embeddings)` 得到）为 Key/Value，做多头交叉注意力，输出 `(B, L, d_llm)`。
3. 将上述输出与 prompt 的 embedding 拼接后送入 LLM，再经 FlattenHead 得到预测。

## 消融设置（去掉重编程）

- **去掉**：`mapping_layer`、`reprogramming_layer`（即不再使用文本原型做交叉注意力）。
- **保留并替换为**：一个线性层 `reprogram_proj: Linear(d_model, d_llm)`，将 Patch Embedding 的输出直接投影到 LLM 的输入维度。
- 后续流程不变：`enc_out = reprogram_proj(enc_out)` 后与 prompt 拼接进 LLM。

即：**时间序列 Patch 特征只做线性映射后直接输入 LLM，不做跨模态语义对齐。**

## 使用方式

### 训练（消融模型）

```bash
bash scripts/Iron_ablation_reprogramming.sh
```

- 与 `Iron.sh` 仅多 `--ablate_reprogramming`，其余超参与数据一致。
-  checkpoint 保存在：`checkpoints/<setting>-iron_ablate_reprogram/`。

### 评估

```bash
bash scripts/Eval_Iron_ablation_reprogramming.sh
```

- 必须带 `--ablate_reprogramming`，且其他参数与训练一致，才能正确找到并加载 `...-iron_ablate_reprogram` 下的 checkpoint。

## 参数说明

| 参数 | 说明 |
|------|------|
| `--ablate_reprogramming` | 启用消融：不建 `mapping_layer` / `reprogramming_layer`，改用 `reprogram_proj`。 |

## 论文表述建议

- **完整模型**：时间序列经 Patch Embedding 后，通过多头交叉注意力与从词表映射得到的文本原型进行融合，再送入 LLM。
- **消融**：去掉重编程模块（无文本原型、无交叉注意力），仅用线性层将 patch 表示映射到 LLM 维度后直接输入 LLM。
- 对比两者在 Vali/Test 上的指标（如 MSE、MAE、RMSE 或 MASE），即可量化「文本原型 + 跨模态对齐」的贡献。
