# 主模型评估细节：语义输出与混淆矩阵

## 1. 从「文本原型」角度看评估输出了什么

评估时，时间推演模块会根据**首窗口**的「历史 + 预测」生成一段**铁路业务语义解释**（文本），包括：

- **【趋势】**：各品种预测期日均相对历史近期的升降或平稳
- **【发运连续性】**：预测期内各品种零发运占比（批次/专列模式等）
- **【品种关系】**：品种间是否同时发运或此有彼无（联合排产/错峰）
- **【预测期总体】**：预测期合计运量及主导品种

该文本会**在终端打印**，并**写入 checkpoint 目录下的 `semantic_output.txt`**，便于单独查看“主模型在评估时输出了什么语义”。

### 如何得到 semantic_output.txt

在项目根目录执行主模型评估（与训练参数一致）：

```bash
bash scripts/Eval_Iron.sh
```

或：

```bash
python run_eval.py --model_id Iron_96_48 --model_comment iron --model TimeLLM ...
```

评估结束后，在对应 checkpoint 目录下查看：

```
checkpoints/long_term_forecast_Iron_96_48_TimeLLM_..._0-iron/semantic_output.txt
```

打开该文件即可看到当前主模型在**首窗口**上的语义解释内容（文本原型输出）。

## 2. 趋势方向混淆矩阵（优雅图示）

将预测期按**前半段 vs 后半段均值**离散为**上升/持平/下降**，统计「真实趋势 × 预测趋势」的样本数，得到 3×3 混淆矩阵并绘图。

### 步骤

**第一步：评估并保存 pred/true**

```bash
cd /home/hesong/Time-LLM
python run_eval.py --save_pred_true
```

（若用 `Eval_Iron.sh`，需在其中的 `run_eval.py` 命令后加上 `--save_pred_true`。）

会在主模型 checkpoint 目录下生成 `pred.npy`、`true.npy`。

**第二步：画混淆矩阵**

```bash
python scripts/plot_confusion_matrix.py
```

默认读取主模型 iron 的 checkpoint 目录；也可指定目录与输出路径：

```bash
python scripts/plot_confusion_matrix.py --ckpt_dir checkpoints/.../0-iron --out docs/confusion_matrix.png
```

生成的图片会保存为 `confusion_matrix.png`（默认在 checkpoint 目录，或 `--out` 指定路径），可直接用于论文或报告。

### 依赖

```bash
pip install matplotlib seaborn
```
