# 辅助头置信度与「系统直接给 0」——面向工业审稿的业务可解释性

## 为什么需要「懂业务」而不是「只会跑分」

在 ESWA 等工业应用背景深厚的期刊，审稿人常担心**「书呆子模型」**：指标好看，但调度员无法直接采用。

- **普通模型**：预测 0.5 吨 → 调度员问「这 0.5 吨我要派车吗？」→ 模型沉默。
- **本框架（TIME-LLM + 辅助头 + 置信度置零）**：预测 0.5 吨，辅助头给出「有发运」置信度仅 10% → **系统直接给出 0**，并写明「调度员可直接采用：无需派车」。

## 三步：从特征到「置信度」再到「系统给 0」

### 1. 特征提取 (Latent Feature Extraction)

历史序列（如过去 96 天运量）经冻结 LLM 编码为高维表示 **h**，蕴含周期性、趋势及业务信号。

### 2. 线性投影 + 概率归一化 (Aux Head)

- 辅助头：`z = W_aux @ h + b_aux`，得到二分类 logits（0=无发运，1=有发运）。
- **Softmax** 得到概率：P(有发运) = softmax(z)[1]。
- 例如 z 偏向「无发运」时，P(有发运) ≈ 0.1 → 语义即：「根据表征，这 48 天里有货概率约 10%，建议当噪声处理。」

### 3. 系统决策：置信度低 → 整窗置 0

- 评估时使用参数 **`--aux_confidence_threshold`**（默认 0.5）。
- 若 **P(有发运) < 阈值**：将该预测窗口的数值预测**整窗置 0**。
- 语义输出中增加 **【系统决策说明】**：主预测存在小值、辅助头置信度仅 X%（< 设定阈值），系统已将该窗口置 0，**调度员可直接采用：无需派车。**

## 为什么 0.1 可信？（训练保证）

辅助头在训练阶段用 **Binary Cross-Entropy** 与主任务联合训练：

- 真实「无发运」时，若输出 P(有发运)=0.1，Loss 小；若输出 0.9，Loss 大，参数会被更新以压低误判。
- 因此推断阶段的「约 10%」有明确的训练依据，而非随意标定。

## 评估脚本中的使用

- `--output_aux_semantic`：输出辅助任务推断与系统决策说明。
- `--aux_confidence_threshold 0.5`：P(有发运)<0.5 的窗口整窗置 0；设为 `0` 则关闭该逻辑。
- 与 `--zero_threshold` 配合：先按辅助头置信度整窗置 0，再对剩余预测做「接近 0 置 0」的数值阈值处理。

这样，**数值预测 + 辅助头置信度 + 系统给 0** 形成完整链条，满足「工业可用、业务可解释」的审稿关切。
