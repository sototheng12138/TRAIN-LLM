# 训练阶段：框架与逻辑（是否有“置信度”概念）

## 1. 训练时的整体框架

- **主任务**：多步预测，损失为 `L_main`（如 ZeroInflated / MSE）。
- **辅助任务**：二分类「是否有发运」，损失为 `L_aux = CrossEntropy(aux_logits, aux_labels)`，加权叠加：  
  **`L = L_main + λ · L_aux`**（`λ = aux_loss_weight`，如 0.1）。
- **数据流**：每个 batch 的 `batch_x` / `batch_y`（含未来段）→ 模型前向 → 得到主预测 + **z**（辅助头 logits）→ 用未来段生成伪标签 → 算主损失和辅助损失 → 反向传播更新参数。

## 2. 训练时用到的变量（只有 z，没有“置信度 C”）

| 变量 | 含义 | 在训练中是否出现 |
|------|------|------------------|
| **z** | 辅助头线性层输出：`z = W_aux @ h + b_aux`，shape `(B*N, 2)` 或 `(B, 2)`，二类 logits | ✅ 有，即 `aux_logits['has_shipment']` |
| **伪标签 c_disp** | 由「未来段是否有发运」推得，0=无发运，1=有发运 | ✅ 有，即 `aux_labels['has_shipment']` |
| **置信度 C** | C = P(有发运) = softmax(z)[1]，0～1 的概率 | ❌ **训练阶段不显式计算、不参与 Loss、不打印** |

也就是说：**训练时只有 logits z 和离散标签，没有“置信度”这个显式变量。**

## 3. 训练阶段的具体逻辑

1. **前向**  
   模型在需要时 `return_aux_repr=True`，返回 `extra['aux_logits']['has_shipment']`，即 **z**（未做 softmax）。

2. **伪标签**  
   `compute_derived_auxiliary_labels(batch_y_future)` 根据「未来段是否任一时步、任一通道有发运」得到 0/1 标签，与 z 的样本一一对应（若 z 是 per-channel 的，标签会按需扩展对齐）。

3. **损失**  
   - 主损失：主预测 vs 真实未来段（ZeroInflated / MSE 等）。  
   - 辅助损失：`criterion_ce(z, aux_labels['has_shipment'])`，即 **CrossEntropyLoss(z, 标签)**。  
   - 总损失：`loss = L_main + aux_loss_weight * L_aux`。

4. **反向传播**  
   对 `loss` 做 `backward()`，梯度会更新辅助头的 `W_aux`、`b_aux` 以及上游表示。  
   CrossEntropy 内部等价于对 z 做 log_softmax 再 NLL，因此是在用 **z** 去拟合标签，从而**隐式**地让「有发运」类的概率变大、「无发运」类的概率变小。  
   也就是说：**训练目标等价于“让 z 对应的 P(有发运) 在标签=1 时变大、在标签=0 时变小”，但训练代码里从不写出 C = softmax(z)。**

## 4. 训练阶段是否有“置信度”概念？

- **有“置信度”的语义**：模型通过 L_aux 学习的是「有发运 vs 无发运」的概率，推理时我们才把 z 变成 C = softmax(z)[1] 并叫它“置信度”。  
  所以从**概念**上，训练确实是在学“将来会用来当置信度”的东西。
- **没有“置信度”的显式实现**：  
  - 训练时**不**算 C = softmax(z)。  
  - 训练时**不**用 C 参与 Loss（只用 z + CrossEntropy）。  
  - 训练时**不**打印或保存 C。  

结论：**训练阶段没有显式的“置信度”变量，只有 z 和伪标签；置信度 C 是推理阶段对 z 做 softmax 后才出现并用于“裁决”的。**

## 5. 和推理阶段的对应关系（便于写进论文）

- **训练**：用 z 和伪标签算 CrossEntropy，让 z 的分布学会区分「有发运 / 无发运」；不出现 C。
- **推理**：用训练好的 z 算 C = P(有发运)，再用 C 与阈值（如 0.5）做「是否整窗置 0」的裁决。

论文里可以这样写：  
“虽然清零动作和置信度阈值是在推理阶段执行的，但**用于裁决的 z（以及由此得到的 C）的判别能力，是在训练阶段通过大量真实业务样本和 BCE/CrossEntropy 严格训练得到的**，而非在推理时拍脑袋设定。”

## 6. 代码位置速查

| 内容 | 位置 |
|------|------|
| 辅助头定义（z 的来源） | `models/TimeLLM.py`：`self.aux_has_shipment = nn.Linear(self.d_ff, 2)`，`aux_logits = self.aux_has_shipment(repr_pooled)` |

| 辅助损失（只用 z，无 C） | `run_main.py`：`criterion_ce(aux_logits['has_shipment'], aux_labels['has_shipment'])`，`loss = loss + aux_w * aux_loss` |
| 推理时 z → C → 裁决 | `run_eval.py`：对 `all_aux_logits` 做 softmax 得 P(有发运)，再与 `aux_confidence_threshold` 比较并整窗置 0 |
