# 评估时「接近 0 置为真 0」零阈值

## 1. 为什么模型可能给出比 1e-5 更大的值？

会。真实为 0 时，模型可能预测出 0.001、0.01 甚至 0.1，原因包括：
- 回归头是连续输出，没有“必须为 0”的约束；
- ZeroInflated 只是加大 0 的损失权重，不会把预测压成精确 0；
- 噪声、容量或历史里“偶尔非零”都会让预测偏大。

所以需要**阈值**：把“小于某数的预测”都视为业务上的 0，阈值要能覆盖这些“本应是 0 但偏大”的预测，又不能把真实有发运的小量误杀。

---

## 2. 阈值怎么设？

- **原则**：比“有发运”时的典型最小量级小一截，又比数值噪声大。设太大容易把真实小发运误置 0，设太小则 0.01、0.1 等仍算残差，全 0 通道上仍吃亏。

- **三种用法**  
  - **固定值**（量纲已知时）：  
    - 逆变换后是万吨级：可用 `1e-4`、`0.01`、甚至 `0.1`（视你认定的“可忽略发运”）。  
    - 逆变换后是 0~1 或小数：用 `1e-4`、`1e-5` 更稳。  
  - **与量纲挂钩（推荐）**：  
    - 使用自动阈值：`--zero_threshold -1`，代码会用「训练集各通道 std 的最小值的 1%」作为阈值，随数据尺度变化。  
  - **先看数据再定**：  
    - 看逆变换后非零发运的典型最小值和全 0 通道上预测的分布，取一个比“最小有发运”小、比“明显是噪声/误报”大的数。

- **建议**：  
  - 有 StandardScaler 时优先用 `--zero_threshold -1`（自动）；  
  - 若自动偏大/偏小，再改用固定值并写进实验配置，便于复现。

---

## 3. 为什么只在评估阶段做？

- **训练阶段不做**：  
  - 若在算损失前把 pred 里“接近 0”置 0，这些位置梯度会被截断（或需要自定义梯度），主损失和 ZeroInflated 的梯度都不再自然；  
  - 训练目标仍是“预测连续值”，用 ZeroInflated 等已经在对 0 加重惩罚，不需要在训练里再改 pred。

- **评估阶段做**：  
  - 只影响“报出来的指标”和下游语义（有无发运、发运天数等），不改变训练过程；  
  - 业务上“先有无再多少”，把接近 0 的预测当 0 更合理，且全 0 通道上不会因 0.01、0.1 等小残差被 ARIMA 占优。

所以：**零阈值只应在评估阶段使用，训练阶段不要对 pred 做截断。**

---

## 4. 为何「加了和没加一样」？

- 逆变换后 `pred` 在**原始量纲**（如万吨、百吨），模型在“无发运”时常预测 10、50、100 等，而不是 1e-5。
- 自动阈值若用 `0.01*min(std)` 可能**过小**（相对这些预测），几乎没元素被置零，MAE 不变。
- **已做**：自动改为 5%*min(std)，且当该值 <0.01 时用「预测绝对值 5% 分位」保底；并**打印本批被置零元素个数**。若运行后看到「被置零元素数=0」，请改用**固定阈值**（如 `1`、`5`、`10`，按你数据量纲）。

## 5. 用法小结

```bash
# 不用零阈值（默认）
python run_eval.py ...

# 自动（5%*min(std)，必要时 5% 分位保底；看终端打印的「被置零元素数」确认是否生效）
python run_eval.py --zero_threshold -1

# 固定阈值：发运量在千/万吨级可试 1、5、10
python run_eval.py --zero_threshold 1
```

实现位置：`run_eval.py` 中逆变换到原始量纲后、计算 MAE/RMSE 前，对 `pred` 做 `|pred|<threshold → 0`，并打印 `[零阈值] threshold=..., 被置零元素数=...`；后续指标与时间推演、有发运天数等均基于该 `pred`。
