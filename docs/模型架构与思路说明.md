# 整体模型架构与设计思路

本文档概括你当前实现的**完整模型架构**与**核心设计思路**，便于写论文、汇报或对外说明。

---

## 一、整体定位

模型是在 **TIME-LLM** 时序预测框架基础上的扩展，面向**零膨胀、多通道**的铁路货运（如铁矿发运）场景，实现：

1. **主任务**：多步、多通道数值预测（发运量等）。
2. **辅助任务**：从同一骨干表征推断**离散事件状态**（如「是否有发运」），**无需人工标注**，与主任务联合训练。
3. **可解释输出**：在推理阶段将数值预测与辅助判定融合为**业务语义**（趋势、连续性、发运状态、具体天数、建议），通过**固定模板**做确定性解码，避免大模型幻觉。

可概括为：**事件驱动的时序预测 + 无标注辅助推断 + 模板化可解释输出**。

## 二、模型架构（数据流）

### 2.1 总览

```
输入（历史序列 + 领域 prompt）
    → 归一化（RevIN 等）
    → Patching（PatchEmbedding）
    → 重编程层（patch 与文本原型对齐到 LLM 空间）
    → 与 Prompt 拼接后送入冻结 LLM
    → 取最后若干 patch 对应隐状态，展平
    → 双头：数值投影头 → 预测序列 Ŷ；辅助头（可选）→ 发运/空闲二分类
    → 逆归一化得到最终预测
```

训练时：主损失（如 MSE）+ λ·辅助损失（BCE，监督信号来自**数据推导的伪标签**）。  
推理时：除数值预测外，可输出辅助判定 + 基于规则的时间推演语义（趋势、连续性、Dispatch Log 等）。

### 2.2 各模块说明

| 模块 | 作用 | 实现要点 |
|------|------|----------|
| **输入与归一化** | 多通道历史序列 `(B, T, C)`，可选时间戳；RevIN 等归一化 | 与任务设定一致（如 seq_len=96, pred_len=48, enc_in=4） |
| **Patching** | 将序列切成 patch，做线性嵌入，得到 patch 序列 | PatchEmbedding 或 MultiScalePatchEmbedding；patch_len, stride 可配置 |
| **Prompt 构建** | 为 LLM 提供领域与任务上下文 | 数据集描述（如 bulk iron ore, zero-inflated）+ 任务说明（预测未来 pred_len 步）+ 可选输入统计（min/max/median/trend/lags） |
| **重编程层** | 把「patch 嵌入」对齐到「LLM 词空间」 | 可学习 1000 个「文本原型」；patch 作为 query，原型作为 key/value，做 cross-attention，输出送入 LLM；可选返回 patch→原型 attention 做可解释分析 |
| **冻结 LLM** | 通用语言骨干，不更新参数 | LLaMA/GPT-2/BERT 等；inputs_embeds = [prompt_emb; 重编程后的 patch_emb]，输出 last_hidden_state |
| **表征聚合** | 取与 patch 对应的隐状态，在通道与时间维做平均 | `block = dec_out[..., -patch_nums:]`，pool 得到 `repr_pooled`（用于辅助头） |
| **数值投影头** | 将聚合后的 patch 隐状态映射为预测序列 | FlattenHead：flatten + Linear → (B, C, pred_len)，即多通道、多步预测 Ŷ |
| **辅助头（可选）** | 从同一 pool 表征做二分类 | `Linear(d_ff, 2)`：0=无发运，1=有发运；仅当 `use_aux_loss=True` 时存在并参与训练 |
| **逆归一化** | 将预测还原到原始量纲 | 与输入归一化配套的 denorm，便于评估与下游语义模块 |

### 2.3 重编程层与「文本原型」

- **文本原型**：从 LLM 词表 embedding 经可学习线性层得到 1000 个「原型向量」，视为可解释的语义锚点。
- **重编程**：patch 嵌入作为 query，原型作为 key/value，做 multi-head cross-attention，输出与 prompt 拼接后进 LLM。这样时序 patch 被「重编程」到语言空间，便于冻结 LLM 理解。
- **可解释**：可导出 patch→原型的 attention 权重，以及原型与词表中 top-k 词的对应关系（如 `prototype_word_weight_matrix.npy`），用于热力图等分析。

## 三、训练思路

### 3.1 主任务损失

- 目标：预测未来 `pred_len` 步、`enc_in` 通道的序列，与真实 `Y` 对齐。
- 损失：默认 MSE；可选 MASE 等。对零膨胀数据，可替换或加权的 Zero-Inflated MSE（代码中主损失接口统一，具体形式可配置）。

### 3.2 辅助任务与「无标注」设计

- **辅助目标**：二分类「本窗口未来段是否存在发运」（0=无，1=有），与主任务**共享骨干 + 重编程 + LLM**，仅多一个轻量级 MLP 头。
- **标签从何而来**：**不依赖人工标注**。在训练时，用**同一 batch 的 ground-truth 未来段** `Y_future` 按规则生成伪标签：
  - `C_disp = 1` 若 `sum(|Y_future| > ε)` 存在（即未来段内任一时间、任一通道非零），否则 `C_disp = 0`。
- **辅助损失**：对辅助头输出的 2 维 logits 做 **CrossEntropy / BCE**，监督信号即上述 `C_disp`。
- **联合目标**：  
  **L_total = L_main + λ·L_aux**  
  其中 `L_main` 为主任务损失（如 MSE），`L_aux` 为辅助分类损失，λ 为超参（如 0.2）。  
  这样既拟合数值，又让同一隐表征具备「是否发运」的判别能力，起到**隐式正则**与**事件感知**作用。

### 3.3 训练流程要点

- 前向：`model(..., return_aux_repr=True)` 得到预测与 `aux_logits`。
- 主损失用 `Ŷ` 与 `Y` 计算；辅助损失用 `aux_logits['has_shipment']` 与 `compute_derived_auxiliary_labels(batch_y_future)` 的 `has_shipment` 计算。
- 只训练：patch 嵌入、重编程层、映射层、数值头、辅助头；**LLM 参数冻结**。

## 四、推理与可解释输出

### 4.1 数值预测

- 与训练相同前向，得到 `Ŷ`，经逆归一化得到原始量纲的多步、多通道预测。
- 用于指标计算（如 MAE、RMSE）以及下游**时间推演模块**的输入。

### 4.2 辅助推断（发运状态 + 置信度）

- 同一前向中取 `aux_logits['has_shipment']`，对首样本做 softmax 得到 P(无发运)、P(有发运)，取 argmax 为 0/1，概率为置信度（如 99%）。
- **「若干天」具体化**：对当前窗口的预测序列 `Ŷ[0]` 按时间步统计「任一道 > ε」的天数，得到「有发运天数」，填入模板，避免模糊表述。

### 4.3 时间推演模块（规则 + 模板，无大模型）

- **输入**：同一窗口的**历史**（逆归一化后）与**预测** `Ŷ`，以及通道名。
- **逻辑**：纯规则与统计，不经过 LLM，不微调。
  - **趋势**：各通道预测期均值 vs 历史近期均值，判升/降/平稳，套模板。
  - **发运连续性**：各通道在预测期内「接近零」的占比，分档（连续/部分无/超一半无），套模板。
  - **品种关系**：通道间二值序列相关，得到同时发运/此消彼长等，套模板。
  - **预测期总体**：总发运量、主导品种占比，套模板。
- **输出**：与辅助判定、原型推理（若有）拼接为一段**业务语义文本**，写入 `semantic_output.txt` 并打印；**全部为确定性模板**，无幻觉。

### 4.4 文本原型推理（可选）

- 若导出并加载了「patch→原型」attention 与原型–词表权重，可对首样本做 top-k 原型及其对应语义词的展示，作为对「模型在看什么语义」的补充说明，与趋势/发运结论相互印证。

## 五、设计思路小结

| 思路 | 体现 |
|------|------|
| **事件驱动** | 辅助头显式建模「是否有发运」等离散事件，与主任务共享表征，推理时给出事件级结论与具体天数。 |
| **无标注辅助** | 辅助标签完全由「未来段是否非零」推导，无需人工标注，易扩展到其他二值/多值事件。 |
| **可解释、零幻觉** | 对外输出为「数值 + 事件状态 + 规则语义」；语义部分由模板与统计生成，不经过 LLM 生成文本，避免幻觉。 |
| **冻结 LLM** | 利用预训练语言模型的表征能力，通过重编程与 prompt 适配时序，不微调 LLM，节省算力、保持稳定性。 |
| **重编程 + 原型** | 时序 patch 与文本原型对齐，既提升预测又便于可解释分析（热力图、top 词等）。 |

整体上，这是一个**以 TIME-LLM 为骨干、以联合训练的无标注辅助头为事件侧、以规则时间推演为可解释输出**的完整架构，适合作为「事件驱动的时序预测 + 可解释决策支持」的系统描述与论文主图依据。
