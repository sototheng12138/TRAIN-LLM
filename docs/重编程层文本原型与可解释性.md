# 重编程层（Reprogramming Layer）与文本原型（Text Prototypes）可解释性

## 1. 结构简述

- **文本原型数量**：`num_tokens = 1000`，即 1000 个可学习的“文本原型”向量。
- **原始词表**：LLM 词表大小 `vocab_size`（如 LLaMA 约 32k）。
- **mapping_layer**：`nn.Linear(vocab_size, num_tokens)`，权重矩阵 `W` 形状为 `(1000, vocab_size)`。
  - `W[i, j]`：第 `i` 个文本原型对第 `j` 个词的线性权重。
  - 前向时：`source_embeddings = mapping_layer(word_embeddings.T).T`，得到 1000 个原型在 LLM 嵌入空间的向量 `(1000, d_llm)`，作为重编程层里 cross-attention 的 key/value。
- **重编程层**：时序 patch 嵌入作 query，上述 1000 个原型作 key/value，做 cross-attention，将时序表示“重编程”到 LLM 空间。

因此，**文本原型 = 词表上的 1000 个线性组合**；权重矩阵 **W** 就是“文本原型与原始词集之间的权重关系矩阵”。

## 2. 如何提取这些细节

在项目根目录执行：

```bash
cd /home/hesong/Time-LLM
python scripts/extract_reprogramming_prototypes.py --ckpt_dir checkpoints/long_term_forecast_Iron_96_48_TimeLLM_..._0-iron
```

不写 `--ckpt_dir` 时，脚本会使用默认主模型 iron 的 checkpoint 目录。

**推荐**：若环境无 LLaMA/transformers/tokenizer（如无网络、无 SentencePiece），可加 **`--weights_only`**，仅从 checkpoint 读取权重并保存矩阵与 top-k（此时 `word` 列为 `id_0`、`id_1` 等，无真实词面）。若需真实词面与 `prototype_embeddings_llm_space.npy`，请在能加载 LLaMA 的环境中不加 `--weights_only` 运行。

可选参数：

- `--top_k 20`：每个原型保存权重量最大的 20 个词（默认 20）。
- `--out_dir <路径>`：输出目录，默认 `ckpt_dir/prototype_export`。
- `--weights_only`：仅读 checkpoint 权重，不加载完整模型（无需 LLaMA/tokenizer）。
- `--max_prototypes N`：只导出前 N 个原型的 top-k 词（如 `--max_prototypes 100` 只导出前 100 个原型，得到更小的 JSON/CSV）。
- `--plot`：额外画一张“原型 × 词表子集”的热力图（需 matplotlib/seaborn）。
- `--plot_top_vocab 200`：热力图中只画权重方差最大的 200 个词（默认 200）。

**得到「token id 对应的具体词」**：若用 `--weights_only`，默认 `word` 列为 `id_2579` 等形式。要得到 LLaMA 的真实 token 字符串（如 `▁the`、`▁value`），有两种方式：

1. **推荐**：在同一次提取时加 **`--decode_words`**（需已安装 `sentencepiece`），则导出的 CSV/JSON 中 `word` 列即为真实 token：
   ```bash
   pip install sentencepiece
   python scripts/extract_reprogramming_prototypes.py --ckpt_dir checkpoints/long_term_forecast_Iron_96_48_TimeLLM_..._0-iron --weights_only --max_prototypes 100 --decode_words
   ```
   `--ckpt_dir` 填**含 `checkpoint` 文件的那一层目录**（即 `prototype_export` 的上一级）。

2. 或事后对已导出的目录运行解码脚本（同样需能加载 LLaMA tokenizer）：
   ```bash
   python scripts/decode_prototype_words.py --export_dir checkpoints/.../prototype_export --max_prototypes 100
   ```
   会生成 `prototype_topk_words_decoded.json` / `.csv`，其中 `word` 为真实 token。

## 3. 输出文件说明（均在 `out_dir` 下）

| 文件 | 说明 |
|------|------|
| **meta.json** | `num_tokens`、`vocab_size`、`weight_matrix_shape`、`weights_only`、`prototype_embeddings_shape`（完整模型时）等。 |
| **prototype_word_weight_matrix.npy** | 权重矩阵 W，形状 `(1000, vocab_size)`，即**文本原型 × 原始词集权重关系矩阵**。 |
| **prototype_embeddings_llm_space.npy** | 仅完整模型时生成。1000 个原型在 LLM 嵌入空间的向量，形状 `(1000, d_llm)`，可用于 t-SNE、相似度等。 |
| **mapping_layer_bias.npy** | mapping 层偏置，形状 `(1000,)`。 |
| **reprogramming_layer_query_projection_weight.npy** | 重编程层 query 投影权重 `(1024, 32)`，将时序 patch 嵌入映射到 attention 空间。 |
| **reprogramming_layer_query_projection_bias.npy** | 同上偏置。 |
| **reprogramming_layer_key_projection_weight.npy** | 重编程层 key 投影 `(1024, 4096)`，作用于 1000 个原型在 LLM 空间的向量。 |
| **reprogramming_layer_key_projection_bias.npy** | 同上偏置。 |
| **reprogramming_layer_value_projection_weight.npy** | 重编程层 value 投影 `(1024, 4096)`。 |
| **reprogramming_layer_value_projection_bias.npy** | 同上偏置。 |
| **reprogramming_layer_out_projection_weight.npy** | 重编程层输出投影 `(4096, 1024)`。 |
| **reprogramming_layer_out_projection_bias.npy** | 同上偏置。 |
| **prototype_topk_words.csv** | 每个原型的 top-k 词：`prototype_id, rank, vocab_id, word, weight`。 |
| **prototype_topk_words.json** | 按原型 id 分组的 top-k 词及权重，便于按原型查阅。 |
| **prototype_word_heatmap.png** | 仅在使用 `--plot` 时生成：W 的子矩阵（前 `plot_top_vocab` 个高方差词）热力图。 |
| **prototype_word_weight_matrix_sub.npy** | 仅在使用 `--plot` 时生成：上述子矩阵。 |
| **heatmap_vocab_subset.txt** | 仅在使用 `--plot` 时生成：子矩阵对应的 `vocab_id` 与词。 |

## 4. 后续可解释性与作图建议

1. **原型–词表关系**
   - 用 `prototype_word_weight_matrix.npy` 画部分原型 × 部分词的热力图（可先按方差选词/原型）。
   - 用 `prototype_topk_words.csv` 或 `.json` 分析每个原型对应的“最相关词”，做表格或词云。

2. **原型在语义空间中的分布**
   - 用 `prototype_embeddings_llm_space.npy` 做 t-SNE/UMAP，看 1000 个原型在 LLM 空间是否成簇、是否有明显语义分工。

3. **与任务的关系**
   - 若后续有“哪些原型对预测重要”的权重（例如从 head 或 attention 反推），可与 `prototype_topk_words` 结合，解释“重要原型对应哪些词”。

4. **热力图**
   - 脚本已支持 `--plot` 生成子矩阵热力图；若要自定义（例如选不同原型/词子集），可读 `prototype_word_weight_matrix.npy` 和 `meta.json` 自行画图。

---

## 5. 与论文/报告表述的对应

- **“主模型 checkpoint 对应输出了多少 text prototypes”**：**1000 个**（`num_tokens=1000`）。
- **“提取学习到的文本原型与原始词汇集之间的权重关系矩阵”**：即 **`prototype_word_weight_matrix.npy`**，形状 `(1000, vocab_size)`；行=原型，列=词，元素=权重。
- 可解释性部分可写：通过该权重矩阵与每个原型的 top-k 词，分析时序表示被映射到了哪些“词级”语义上。

## 6. 重编程层细节汇总（便于做图与可解释性）

| 项目 | 值 / 文件 |
|------|-----------|
| 文本原型数量 | **1000** |
| 原始词表大小 | **32000**（LLaMA） |
| 原型–词表权重矩阵 | `prototype_word_weight_matrix.npy`，形状 `(1000, 32000)` |
| 每个原型的 top-k 词及权重 | `prototype_topk_words.csv` / `prototype_topk_words.json` |
| 原型在 LLM 空间的向量（t-SNE/相似度用） | `prototype_embeddings_llm_space.npy`（需完整模型，非 `--weights_only`） |
| mapping 层偏置 | `mapping_layer_bias.npy`，形状 `(1000,)` |
| 重编程层 query/key/value/out 投影 | 对应 `reprogramming_layer_*_projection_*.npy` |

作图建议：用 `prototype_word_weight_matrix.npy` 画原型×词子集热力图；用 `prototype_topk_words.json` 做词云或表格；若有 `prototype_embeddings_llm_space.npy` 可做 1000 个原型的 t-SNE/UMAP。热力图也可由脚本直接生成：安装 `matplotlib`、`seaborn` 后加 `--plot` 重新运行即可。
