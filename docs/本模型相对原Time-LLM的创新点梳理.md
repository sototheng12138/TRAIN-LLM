# 本模型相对原 Time-LLM 的创新点梳理（供论文撰写）

本文档基于当前代码与设定，梳理相对原版 Time-LLM 的**可写创新点**，便于在论文中组织贡献陈述。

---

## 一、您已归纳的两点

### 1. Loss 函数设计

- **ZeroInflatedLoss**：针对铁路货运数据的**零膨胀分布**（40%–86% 不规则零值），对「目标为 0」的样本施加更大惩罚权重（`zero_weight`，默认 2.0），迫使模型更好地区分「零运量日」与「高运量日」，避免对零值欠拟合或过平滑。
- **MASE_loss_multivariate**：多变量 MASE，以历史 in-sample 的步长差分尺度为分母做缩放误差；并对零膨胀/近常数序列做 **分母下界（eps）** 稳定训练。支持与 ZeroInflated 一样作为可选训练目标（`--loss MASE`），原 Time-LLM 默认使用 MSE。
- **论文可写角度**：面向零膨胀多通道时序的损失设计，使模型在「有发运/无发运」与多通道输出场景下更稳、更贴合业务需求。本实验为多通道同时预测，未显式建模通道间协同规律。

### 2. 铁路业务事件推演模块

- **模块位置与接口**：`utils/railway_semantic.py` 中的 `generate_railway_semantic(history, pred, col_names, pred_len)`，在**评估阶段**基于「历史 + 预测」融合序列做后处理，不参与训练。
- **功能**：对预测结果做**时间推演与业务解读**，输出面向铁路调度的中文语义，包括：  
  **【趋势】** 各品种预测期日均相对历史近期的升降与平稳；  
  **【发运连续性】** 预测期内零发运占比（批次/专列模式、部分无发运、较连续）；  
  **【品种关系】** 品种间在预测期内的并发/替代关系（由时间推演模块基于预测序列后验统计得出，非模型显式学习的通道间依赖）；  
  **【预测期总体】** 预测期合计发运量及主导品种，并给出车皮与装卸建议。
- **论文可写角度**：在 Time-LLM 的「预测输出」之上，增加**领域可解释层**，将数值预测转化为调度可用的语义事件描述，形成「预测 → 事件推演」的闭环。

## 二、可补充的创新/贡献点

### 3. 领域定制 Prompt 与任务设定

- **Prompt**：`dataset/prompt_bank/iron.txt` 针对铁路货运四类（铁矿石、铁矿砂、铁矿粉、铁精矿粉）、零值含义（停运/缺记）、零膨胀与「区分零日与高量日、避免过平滑」等做**领域化任务描述**，而非原版 ETT 等通用描述。
- **论文可写角度**：将 Time-LLM 的 in-context 能力与**铁路货运场景**对齐，通过领域 prompt 约束模型对零值、多品种、调度需求的理解，属于「任务与 prompt 设计」上的适配与创新。

### 4. 零膨胀多通道场景下的完整建模链路

- **数据与设定**：2023–2025 年铁路货运量、多通道、高比例零值、7:1:2 划分、RevIN、多步预测（如 96→48）。
- **论文可写角度**：系统性地将 Time-LLM 应用于**零膨胀、多通道铁路货运**场景（四通道联合输入与输出，未显式建模通道间协同），并在数据划分、归一化、评估指标（如 MAE/RMSE 及标准化口径）上与基线一致，形成可复现的基准与对比框架，可作为「应用与方法适配」的贡献。

### 5. 可解释性链路：从重编程层到业务语义

- **重编程层分析**：从 checkpoint 提取 **mapping 层权重**（文本原型 × 词表），得到 1000 个原型在词表上的分布；并针对**业务相关词集**（时间/预测步、货运量/统计、领域/业务）做热力图可视化。
- **与语义模块的衔接**：预测结果既用于**铁路事件推演**（趋势、连续性、品种关系、总量与主导品种），又与**原型–词表可视化**一起，形成「**数值预测 → 业务事件文本 + 原型–词表可解释性**」的完整链路。
- **论文可写角度**：在原 Time-LLM 上增加**可解释性闭环**：不仅给出预测，还给出「为何如此预测」的语义（事件推演）与「原型–业务词」的权重视图，便于调度理解与模型审计。

### 6. 多通道预测与品种关系的后验描述

- **模型侧**：四通道联合输入与联合预测，共享时序表示与重编程层；**未显式建模通道间协同或依赖关系**（即非「多元预测」意义上的协同规律建模）。
- **语义侧**：事件推演模块在**后处理**中基于预测序列统计品种间「并发/替代」关系（如预测期内非零日重叠与相关），并输出为自然语言，属规则推演而非模型学习的通道间结构。
- **论文可写角度**：在多通道 Time-LLM 基础上，将**品种关系**作为可解释输出的一部分（由时间推演模块基于预测结果后验得出），服务于排产与车皮调度；与「显式建模多元协同规律」的方法相区分。
