# 独立通道 vs 多元预测：当前设定与可选改法

## 一、当前 Iron.sh 的设定：独立通道

- **数据侧**：`scripts/Iron.sh` 未加 `--multivariate`，故 `Dataset_Custom_Iron` 使用默认 `return_multivariate=False`。
  - 每个样本 = **一个时间窗口 × 一个通道**：`seq_x` 形状为 `(seq_len, 1)`，`seq_y` 为 `(label_len+pred_len, 1)`。
  - 样本总数为 `tot_len × enc_in`（每个窗口拆成 4 个独立样本，每通道一个）。
- **模型侧**：TimeLLM 在 `forecast()` 内将输入 `(B, T, N)` 做 `reshape(B*N, T, 1)`，即把每个通道当作独立的一条序列处理，**通道之间无信息交互**。
- **结论**：当前是**独立通道**：数据按通道拆样本，模型按通道拆 batch，不建模通道间协同/依赖（非「多元预测」意义上的协同规律）。

---

## 二、若希望改为「多元」：两种层次

### 2.1 仅改数据：加 `--multivariate`（联合样本，模型仍按通道独立）

- 在 `Iron.sh` 的 `run_main.py` 参数中增加一行：`--multivariate`。
- 效果：
  - 每个样本 = **一个时间窗口 × 全部通道**：`seq_x` 为 `(seq_len, 4)`，`seq_y` 为 `(label_len+pred_len, 4)`；样本数变为 `tot_len`（不再乘 enc_in）。
  - 训练时一个 batch 内是「同一窗口的四通道」一起进模型。
- **但**：TimeLLM 仍会执行 `reshape(B*N, T, 1)`，即把同一窗口的 4 个通道拆成 4 条独立序列分别前向，**仍不建模通道间依赖**，只是数据组织方式从「按通道采样」变为「按窗口采样、每窗口 4 通道」。若要做基线对比（如与「按窗口联合输入」的 Autoformer 对齐），可在此层次加 `--multivariate`。

### 2.2 真正建模协同规律：需改模型

- 若希望模型**显式利用通道间依赖**（多元预测 / 协同规律），需要：
  - **不再**在骨干内把 `(B, T, N)` 压成 `(B*N, T, 1)`；
  - 在 patch 嵌入或 LLM 之前/之后保留通道维，并引入**通道间交互**（例如：在 patch 维度上保留 channel，或加一层 channel-wise attention / MLP mixer），使不同通道的表示能互相影响。
- 这属于模型结构改动，需改 `models/TimeLLM.py` 中的 `forecast()` 及 patch 相关逻辑，并重新调参与实验。

---

## 三、小结

| 项目         | 当前（Iron.sh 默认）     | 仅加 `--multivariate`     | 真正多元（协同）     |
|--------------|--------------------------|----------------------------|----------------------|
| 数据样本     | 每样本 1 通道            | 每样本 4 通道（同窗口）    | 同左                 |
| 模型是否用 N | 否，reshape 成 B*N 条单通道 | 否，同上                   | 是，保留 N 并做通道交互 |
| 是否建模协同 | 否                       | 否                         | 是                   |

论文/贡献中若强调「未显式建模通道间协同」，与当前实现一致。

## 四、多元协同脚本（已实现）

- **训练**：`scripts/Iron_multivariate.sh`  
  - 使用 `--multivariate`（每样本同窗口四通道）+ `--channel_mixing`（TimeLLM 内对 (B,T,N) 做线性混合后再 reshape）。  
  - checkpoint 目录带 `model_comment=iron_multivariate`，与 `Iron.sh` 的 `iron` 区分。
- **评估**：参数需与训练一致，例如  
  `python run_eval.py --model_comment iron_multivariate --multivariate --channel_mixing`  
  若需辅助语义输出可加 `--output_aux_semantic`。
- **模型改动**：`models/TimeLLM.py` 中增加 `channel_mixer = nn.Linear(enc_in, enc_in)`，在 `forecast()` 内于 `reshape(B*N, T, 1)` 前对 `x_enc (B,T,N)` 做一次线性变换，使各通道含其他通道信息。
